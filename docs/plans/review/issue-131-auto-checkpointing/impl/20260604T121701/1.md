## Change Summary
- Opt-in `auto_checkpoint` config block (schema + Pydantic + accept/reject parity fixtures); defaults applied in the orchestrator, NotNone-wrapped inner fields.
- `CheckpointScheduler` pure-logic module: cadence timer (first fire one interval after start; advances from completion → no catch-up storm), surgical retention ring, one-shot terminal trigger with restart dedup, best-effort failure isolation (no retry storm; never crashes loop), atomic in-dir temp-file write, sanitize+sha256 collision-resistant filename prefix.
- Loop fires periodic each iteration + terminal on observed `terminated` state, gated so disabled deployments pay no extra wire read; single exit-path terminal re-check.
- CLI `--auto-checkpoint-dir`/env; fail-fast at startup on enabled-without-admin-token or unusable dest dir; admin-bearer export client (export endpoint is admin-gated, plan D4).
- Compose bind-mount + setup-experiment dir; smoke-auto-checkpoint.sh (validated locally PASS: 3 periodic + 1 terminal) + CI job.
- Deferrals filed: #292 (multi-experiment), #293 (server-side on-terminate hook), #294 (empty git bundle under Compose).
