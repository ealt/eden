## Change Summary

- **86 v1 scenarios + 1 meta** all pass against the reference IUT (`uv run pytest -q conformance/` exits 0 in ~50s).
- **Citation check** — every scenario docstring's first line cites a normative MUST in `spec/v0/<chapter>.md §<sec>`; tool walks all scenarios and validates the citation resolves. CI gates this.
- **Vocabulary closure** — every chapter-7 §7 `eden://error/<name>` entry is observed at least once across the suite (incl. new duplicate-create scenario for `already-exists`); no other types appear.
- **Self-validation** — meta scenario runs subprocess pytest 3× (control / broken-adapter / hit-counter) to prove the harness has teeth.
- **Layer discipline** — directionality grep gates that `harness/`, `_meta/`, and `scenarios/` never import `reference/` packages; only `adapters/reference/` may.
- **Reference adapter** uses `secrets.token_hex` (not `token_urlsafe`) because the latter occasionally produces tokens starting with `-` that argparse interprets as a flag.
- **Subscribe timeout** is set to 2.0s on the reference adapter so the timeout-path subscribe scenario completes quickly; chapter 7 §6.2's 30s is a SHOULD, so this is conformant.
