**Findings**

1. The manifest semantics still resurrect the terminated-import contradiction. `requires_credential_reissue` says that when the flag is `true`, the importer MUST reissue credentials and the experiment MUST start in `state="running"` [12b:409](/Users/ericalt/Documents/eden/docs/plans/eden-phase-12b-portable-checkpoints.md:409). But §3.6 now says import preserves `experiment.json` state verbatim, including `terminated` [12b:487](/Users/ericalt/Documents/eden/docs/plans/eden-phase-12b-portable-checkpoints.md:487), [12b:493](/Users/ericalt/Documents/eden/docs/plans/eden-phase-12b-portable-checkpoints.md:493), and §3.8 repeats that terminated experiments import as terminated [12b:587](/Users/ericalt/Documents/eden/docs/plans/eden-phase-12b-portable-checkpoints.md:587). Since worker-bearing checkpoints set `requires_credential_reissue` [12b:454](/Users/ericalt/Documents/eden/docs/plans/eden-phase-12b-portable-checkpoints.md:454), the plan still requires the same import to be both `running` and `terminated`.

2. The chosen import endpoint still conflicts with chapter 07’s global experiment-scoping rule. The existing wire binding says every normative path below `/v0/` begins with `experiments/{experiment_id}/` and carries a matching `X-Eden-Experiment-Id` header [spec/v0/07-wire-protocol.md:23](/Users/ericalt/Documents/eden/spec/v0/07-wire-protocol.md:23). The plan still uses `POST /v0/checkpoints/import` [12b:431](/Users/ericalt/Documents/eden/docs/plans/eden-phase-12b-portable-checkpoints.md:431), and the spec edits only say “add §8 checkpoint operations” [12b:758](/Users/ericalt/Documents/eden/docs/plans/eden-phase-12b-portable-checkpoints.md:758). Unless §1.3 is explicitly amended to carve out global checkpoint endpoints, the new import path is not valid under the current binding contract.

3. §7.2 still contains the old direct-streaming implementation story and now contradicts the revised Decision 8 / §3.7 contract. Decision 8 says export is “materialize the artifact” server-side before delivery [12b:190](/Users/ericalt/Documents/eden/docs/plans/eden-phase-12b-portable-checkpoints.md:190), and §3.7 shows the temp-file boundary explicitly [12b:514](/Users/ericalt/Documents/eden/docs/plans/eden-phase-12b-portable-checkpoints.md:514), [12b:530](/Users/ericalt/Documents/eden/docs/plans/eden-phase-12b-portable-checkpoints.md:530). But §7.2 still says the reference impl uses `tarfile.open(..., 'w|')`, yields chunks directly to the HTTP response, and holds the snapshot open for the duration of the stream [12b:962](/Users/ericalt/Documents/eden/docs/plans/eden-phase-12b-portable-checkpoints.md:962), [12b:964](/Users/ericalt/Documents/eden/docs/plans/eden-phase-12b-portable-checkpoints.md:964). That section now describes the superseded design, not the one the plan settled on.

**Overall Assessment**

Round 3 fixes the retry-surface and event-shape issues, but I would still stop at feasibility. The plan is close, yet it still has three internal contract collisions: manifest-state semantics, wire-path scoping, and the stale §7.2 streaming narrative. Once those are reconciled, I’d move on to alternatives.
