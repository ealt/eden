## Change Summary
- New `eden_service_common.artifacts`: `entity_artifact_dir(producer, entity_id)` + `ArtifactNaming` (`idea_naming()` / `submission_naming(stem, headline)`) + `predict_artifact_uri` / `write_artifact_bundle` taking `(target_dir, naming)`; predict and write share one name-derivation path. Read-side helpers (`is_bundle_uri`/`read_bundle_*`) kept in web-UI; write-side re-exported from `_common`.
- No-overwrite (chapter-8 ¬ß5.4): final materialization is an exclusive `os.link(tmp, final)` that raises `FileExistsError` rather than clobbering ‚Äî caller-independent.
- Ideator text filename unified on `content.md` across write path, predict collision-check, and the inline-render reader constant (`IDEA_BUNDLE_HEADLINE`).
- CLI write-side URI stamp changed from basename-only to full nested relative path (¬ßD.4 trap); read-side translate was already nested-safe.
- DEVIATION: the web-UI executor byte-writer (`_maybe_bundle_executor_artifact`, landed by #212 after the plan was written) was routed hierarchically rather than left flat, to keep all four writers consistent.
- Spec: layout documented as a reference-binding detail only (worker-host-subprocess ¬ß10); core protocol unchanged (08-storage ¬ß5.1 stays "implementation-defined").
