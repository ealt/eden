## Change summary

- The per-scenario `iut` fixture (function-scoped) spawns a fresh in-memory task-store-server + control-plane subprocess on random ports; the suite is boot-bound, not compute-bound, and has no inherently-serial long pole — ideal for xdist.
- `session_observed_problem_types` (session-scoped stash, populated by every scenario's `WireClient`) is the ONLY cross-test accumulator. Only `test_error_vocabulary.py` reads it. Under xdist it is per-worker, so the "every core type observed at least once" direction fails on a naive `-n auto`.
- Fix: aggregate each worker's observed types to the controller (`pytest_sessionfinish` worker → `workeroutput`; `pytest_testnodedown` controller → union stash) and assert BOTH closure directions over the union at controller `pytest_sessionfinish`, setting `session.exitstatus = TESTS_FAILED` on violation (without clobbering a pre-existing failure). The two scenario tests skip on workers (`hasattr(config, "workerinput")`), assert in-process under serial.
