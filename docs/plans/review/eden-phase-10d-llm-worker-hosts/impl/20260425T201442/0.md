## Change Summary

- Three new helpers in `eden_service_common`:
  `TaskWorktree` / `sweep_host_worktrees` / `subprocess_runner`
  (Popen with `start_new_session=True`, stdout queue, stderr→logger,
  SIGTERM→SIGKILL ladder) / `experiment_config` (load + extract
  `*_command` from `model_extra`).
- Each of planner / implementer / evaluator hosts gains a `--mode
  {scripted,subprocess}` flag with subprocess-mode CLI flags.
  Default is unchanged (`scripted`).
- Implementer flow honors spec §3.2 step 1: pre-Phase-1 ref-collision
  guard → `Store.create_trial(status="starting")` → worktree create
  → run subprocess → validate commit_sha + reachability →
  `repo.create_ref` → submit with retry-before-orphan + read-back.
- Evaluator flow: claim → worktree at `trial.commit_sha` → run
  subprocess → validate metrics via `Store.validate_metrics` →
  submit (with read-back).
- Planner subprocess uses a long-running JSON-line protocol; loop
  respawns the subprocess on crash and continues.
- Cross-host isolation: each host writes worktrees under
  `<worktrees-dir>/<container_hostname>/<task_id>/`;
  startup-time `sweep_host_worktrees` uses path-scoped
  `git worktree remove --force` (never repo-global
  `git worktree prune`).
- Failure context goes to host log only (chapter-3 submission shapes
  carry no `description` field).
- Fixture experiment grows three deterministic command scripts so
  the new `compose-smoke-subprocess` CI job can run end-to-end
  without LLM secrets.
- Compose plumbing is a non-invasive overlay
  `compose.subprocess.yaml` layered via `compose -f compose.yaml -f
  compose.subprocess.yaml`.
- Container isolation of the user's command is **deferred** to a
  10d follow-up sub-chunk per the plan's roadmap delta.
