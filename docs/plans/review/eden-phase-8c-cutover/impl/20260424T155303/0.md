## Change Summary

- `eden-dispatch.run_experiment` and the in-process Phase 5/7b end-to-end tests are gone. The dispatch package's public surface narrows to `run_orchestrator_iteration` plus the three scripted workers. No deprecation shim — the package is at version `0.0.1` with no external consumers (compatibility policy explicit in the plan).
- The lifecycle-reconstruction invariant from the deleted `test_event_log_reconstructs_full_lifecycle` is preserved as a `_reconstruct_lifecycle` fold inside the existing `pytest.mark.e2e` real-subprocess test in `reference/services/orchestrator/tests/test_e2e.py`. The fold runs against `SqliteStore.read_range()` reopened after the orchestrator/worker subprocesses are torn down — the same inspection path the test already used. Three plan tasks → three proposals → three trials → 9 task lifecycle entries; assertion uses 9 (not 7 from the in-process version, which seeded only one plan task).
- The malformed-success → `validation_error` routing case in `test_store_hardening` is rewritten as a direct unit test: ready a proposal, create+claim+submit an implement task with `commit_sha=None`, run `run_orchestrator_iteration`, assert `state == failed` and a `task.failed(reason=validation_error)` event. No scripted workers, no `run_experiment`.
- Verification gates all green locally: ruff clean, pyright 0/0/0, pytest 409 passed, markdownlint clean, JSON Schema metaschema clean. The two unresolved §-refs reported by `spec-xref-check.py` (`spec/v0/README.md` lines 16 against §3.4 and §1.1) are pre-existing on `main` and unrelated to this branch.
- Live-source grep gate: `grep -rn "run_experiment" reference/` returns zero hits. Historical narration in `AGENTS.md` and `docs/roadmap.md` is preserved in past tense by design (per the grep-gate scope discussion in the plan).
