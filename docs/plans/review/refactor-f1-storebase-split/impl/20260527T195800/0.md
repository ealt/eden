## Change summary

- 8 mixins each inherit `_StoreCore`; `_StoreBase` flattens the mixin
  MRO. Composite-commit invariant preserved: each public method builds
  one `_Tx` and calls `self._apply_commit(tx)` once; `_Tx` field list
  and per-backend walk order are byte-identical (backends untouched).
- Two `_StoreCore` abstract stubs (`resolve_worker_in_group`,
  `_validate_evaluation`) shadowed by their owning mixin (`_GroupOps`,
  `_ExperimentOps`); module-load `assert _StoreBase.__mro__[1:10]`
  guard enforces ordering.
- `_reseed_default_event_counter` `__func__` comparison updated to
  `_StoreCore._default_event_id` (decision #1b).

## Plan deviations (all surfaced in PR #232 + CHANGELOG)

1. `_ValidationOps` mixin rejected — validators co-locate with consumers
   (plan §D.3).
2. L-O: `reassign_task` extracted verbatim (67 LEN / CC 6, already under
   threshold; §D.6 warns against over-splitting). Closes L-O.
3. `_validated_update` re-export retired (dead `# noqa` shim under
   BASE.md no-shim posture); 4 eden-git test sites repointed to
   `_ops._helpers`.
4. `export_checkpoint`/`import_checkpoint` `cast` `self` to `_StoreBase`
   (cross-mixin; plan §8.2 option b) — `_checkpoint` needs the full
   composite surface (core primitives + worker credential helpers).
