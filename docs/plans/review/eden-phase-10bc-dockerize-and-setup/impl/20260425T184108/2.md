# Implementation Review: eden-phase-10bc-dockerize-and-setup — Round 2

## Round 1 issues addressed

- **Risk — task-store-server healthcheck shell-interpolation hazard.**
  Both healthchecks (`task-store-server` and `web-ui`) switched from `CMD-SHELL` to `CMD` argv form in [`compose.yaml`](../../../../../reference/compose/compose.yaml). Compose still interpolates the env-file values exactly once at compose-load time, but no shell re-parses them — quotes / spaces / shell metacharacters in the token or experiment id can no longer break or alter the command.

- **Nit — printed next-step uses hardcoded `.env`.**
  [`setup-experiment.sh`](../../../../../reference/scripts/setup-experiment/setup-experiment.sh) now echoes the actual `${ENV_FILE}` path in both "Next steps" lines, so a custom `--env-file <path>` doesn't print stale guidance.

- **Nit — `test_reopen_inherits_persisted_metrics_schema` didn't validate inheritance.**
  Strengthened in [`test_postgres_store.py`](../../../../../reference/packages/eden-storage/tests/test_postgres_store.py): now exercises `validate_metrics()` after reopen — an unknown metric name raises `InvalidPrecondition`, the persisted name validates fine. A reopened store that failed to inherit the schema would accept the unknown name and fail this assertion.

- **Nit — README sqlite path examples wrong.**
  [`task-store-server/README.md`](../../../../../reference/services/task-store-server/README.md) and [`web-ui/README.md`](../../../../../reference/services/web-ui/README.md) updated to use a bare absolute path (`/tmp/eden.sqlite`) — unambiguous and matches what `build_store()` actually does. The task-store-server README also now documents the `sqlite:////<absolute-path>` four-slash form for readers who prefer the explicit URL scheme.
