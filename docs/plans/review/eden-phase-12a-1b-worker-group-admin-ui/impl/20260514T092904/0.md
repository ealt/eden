## Change Summary

- New `/admin/workers/` + `/admin/workers/<id>/` routes implementing list / register / detail / reissue. Token-display page renders 200 HTML (not 303) with `Cache-Control: no-store`; idempotent re-register on an existing worker renders the same template with "no new token was issued" banner and no token.
- New `/admin/groups/` + `/admin/groups/<id>/` routes implementing list / register / detail / add-member / remove-member / delete. Transitive worker closure walked client-side via DFS over `read_group` with cycle-safe visited set and depth/breadth caps (10/1000).
- `?worker=<id>` filter extension on existing `/admin/tasks/` and `/admin/variants/` pages; client-side post-filter against `claim/submitted_by/created_by` (tasks) and `executed_by/evaluated_by` (variants); invalid value yields empty rowset per chunk-9e `_INVALID_FILTER` discipline.
- `make_app` gains a `admin_store: Store | None` keyword; `cli.py` builds a second `StoreClient` bearing `admin:<admin_token>` when `--admin-token` / `$EDEN_ADMIN_TOKEN` is set; otherwise `admin_store=None` and mutation controls render disabled.
- Posture-D startup guard added to `cli.py`: probes `store.whoami()` after bootstrap; exits non-zero on `Unauthorized` to prevent a silently-broken auth-enabled-but-no-bearer deployment.
- Reserved-identifier + grammar discipline enforced client-side before the wire is touched; defensive backstop on `ReservedIdentifier` / `InvalidPrecondition` / `BadRequest` from the wire.
- Closed banner-key allowlists per module (`_WORKER_OUTCOMES`, `_GROUP_OUTCOMES`); unknown keys render no banner.
