## Change summary

- **One spec edit** — informative §12 documenting the reference-only auth namespace `eden://reference-error/…` and the constraint that any auth scheme MUST NOT overload the normative `eden://error/…` types.
- **One refactor** — `run_experiment` split so the standalone orchestrator drives only the finalize/dispatch/integrate half via the now-public `run_orchestrator_iteration`.
- **One auth extension** — server middleware + client header + `Unauthorized` exception, all kept structurally separate from the normative vocabulary (separate `_REF_TYPE_BY_EXC` table, separate base class, separate envelope helper).
- **Five service entrypoints** — task-store-server (uvicorn), orchestrator (StoreClient + Integrator + dispatch loop), planner / implementer / evaluator hosts (StoreClient + ScriptedX). Each has its own `pyproject.toml`, `__main__.py`, CLI module, and runtime module.
- **Shared scaffolding** — JSON-line logger that includes service+experiment context, SIGTERM/SIGINT stop flag, readiness probe (`wait_for_task_store`), common argparse flags, scripted `plan_fn`/`implement_fn`/`evaluate_fn` profiles (the implementer writes real git commits via `GitRepo.commit_tree` honoring `proposal.parent_commits` for both single-parent and merge shapes), and a `seed_bare_repo` helper.
- **Real-subprocess E2E** — `test_e2e.py` forks all five processes on an ephemeral port (read from the task-store-server's stdout announcement), drives a 3-trial experiment to quiescence, and asserts on the SQLite DB + bare repo. Tears down with SIGTERM → 5s wait → SIGKILL fallback. Captures every subprocess's stderr on failure. Test runs in ~3s when healthy.
