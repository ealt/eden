## Change Summary

- Added the evaluator module under `/evaluator/*` (list / claim /
  draft / submit), mounted unconditionally; the trial_id is
  read from `task.payload.trial_id` at claim time and stashed in
  an in-process `_CLAIMS` dict (per-(csrf, task_id) isolation).
- Generalized the chunk-9c `read_proposal_rationale` trust
  boundary into `_read_inline_artifact(uri, artifacts_dir)` and
  added `read_trial_artifact` for the trial-side surface.
  `read_proposal_rationale` is now a thin wrapper.
- New `parse_evaluate_form` validates per-metric types per the
  spec (`integer` accepts wire-legal `1.0`, `real` rejects non-
  finite, text non-empty after strip, `success` requires ≥ 1
  metric), with closed-set status validation and accumulated
  per-field errors.
- `_retry_submit_with_readback` differs from chunk 9c: `WrongToken`
  / `ConflictingResubmission` short-circuit; `InvalidPrecondition`
  re-renders the form (fixable); `IllegalTransition` and other
  transport-shaped exceptions feed the read-back arm. The read-
  back uses `read_task` + `read_submission` +
  `submissions_equivalent` to distinguish "we won, response
  lost" from "lost the race" from "claim still ours".
- 56 new tests (20 routes + 5 flow + 12 partial-write recovery +
  18 security + 1 e2e). Full suite: 553 unit + 5 e2e green.

## Round 1 — fixes

- **Bug fixed.** [evaluator.py:218–238](reference/services/web-ui/src/eden_web_ui/routes/evaluator.py): the submit handler now collects every submitted `metric.*` field (not just declared schema keys) and forwards them all to `parse_evaluate_form`, so a hand-crafted POST with `metric.ghost` is rejected. Default declared keys to `""` so the parser still sees blanks for the omit-on-empty branch. Companion change in [forms.py](reference/services/web-ui/src/eden_web_ui/forms.py): unknown keys surface as `add_overall(...)` rather than `add(0, name, ...)` so the rejection appears on the rendered form (the field doesn't exist on the page to render an inline error against).
- **Risk fixed.** Read paths in [evaluator.py](reference/services/web-ui/src/eden_web_ui/routes/evaluator.py): `list_pending` (L82-87), `claim` (L114-122), `draft_form` (L156-162), and `submit` pre-reads (L213-219) now also catch transport-shaped `Exception` (the StoreClient surface) and route them through the existing wire-error UX (banner / 502) instead of letting them surface as 500.
