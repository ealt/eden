## Change Summary

- New `termination_policy` discriminated-union block in the schema + `TerminationPolicyConfig` Pydantic union (5 kinds) + cross-field rule "required when dispatch_mode.termination == auto" enforced on BOTH schema (top-level allOf-if-then) and Pydantic (model_validator) sides.
- New `DurationStr` validated-string alias (string validated by TypeAdapter(timedelta) + positivity) with a matching `duration` FormatChecker handler that reuses the same validator for bit-for-bit parity.
- Three scalar fields (`max_quiescent_iterations` ge 2, `*_task_deadline` gt 0).
- New `build_termination_policy(config)` factory in eden-dispatch; removed `env_max_variants_policy()` + EDEN_TERMINATION_MAX_VARIANTS bridge.
- Orchestrator single-experiment branch reads termination_policy + max_quiescent_iterations from config (CLI flags retained for multi-experiment mode pending #214; non-default flag -> startup WARN). Worker hosts drop their deadline flags and read from config in subprocess mode.
- Compose + .env.example drop the retired flags/env; all 7 smokes inject max_quiescent_iterations: 30.
- Spec prose (02 §2.4, 03 §6.2/intro), user-guide, experiment-lifecycle playbook (module:callable -> YAML recipes), glossary, CHANGELOG, roadmap.
