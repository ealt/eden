# Implementation Review: issue-157-cli-flags-to-config

## Overall assessment

The implementation is mostly aligned with plan §3.1-§3.7. The schema/Pydantic work, `DurationStr`/`FormatChecker` parity, retired compose/env cleanup, worker-host deadline wiring, naming discipline, and CHANGELOG deferral tracking all look sound. I found one behavioral regression on the orchestrator multi-experiment path and one operator-doc miss.

## Findings

1. **MAJOR** — Multi-experiment orchestrator startup now trips the new single-experiment `termination_policy` validation before it reaches the still-supported CLI path.
   - [reference/services/orchestrator/src/eden_orchestrator/cli.py](/Users/ericalt/Documents/eden-worktrees/impl-issue-157-cli-flags-to-config/reference/services/orchestrator/src/eden_orchestrator/cli.py:362) loads and validates `args.experiment_config` before branching on `args.control_plane_url`, and [main](/Users/ericalt/Documents/eden-worktrees/impl-issue-157-cli-flags-to-config/reference/services/orchestrator/src/eden_orchestrator/cli.py:366) only reaches `_resolve_termination_policy(...)` afterward.
   - That eagerly applies the new `ExperimentConfig` cross-field rule everywhere: `dispatch_mode.termination == "auto"` now requires `termination_policy`, even in control-plane / multi-experiment mode.
   - The plan explicitly narrows this chunk to the single-experiment orchestrator path and keeps `--termination-policy` / `--max-quiescent-iterations` as the multi-experiment source of truth until [#214](https://github.com/ealt/eden/issues/214). With the current code, a multi-experiment deployment that still relies on the CLI flag and has `dispatch_mode.termination: auto` but no `termination_policy` in the bootstrap YAML exits during config-load before it can use the CLI fallback.
   - I reproduced the underlying failure directly: `ExperimentConfig.model_validate(...)` raises `ValidationError` for `dispatch_mode.termination == 'auto'` without `termination_policy`.
   - Fix shape: defer `load_experiment_config()` / `build_policy(config.ideation_policy)` until after the single-vs-multi branch, or otherwise avoid applying the single-experiment-only termination-policy requirement on the multi-experiment path until #214 lands.

2. **MINOR** — `docs/user-guide.md` still sends operators to the retired `EDEN_MAX_QUIESCENT_ITERATIONS` env var.
   - [docs/user-guide.md](/Users/ericalt/Documents/eden-worktrees/impl-issue-157-cli-flags-to-config/docs/user-guide.md:38) still says to edit `EDEN_MAX_QUIESCENT_ITERATIONS` in `.env`.
   - The quiescence section at [docs/user-guide.md](/Users/ericalt/Documents/eden-worktrees/impl-issue-157-cli-flags-to-config/docs/user-guide.md:190) still instructs operators to set `EDEN_MAX_QUIESCENT_ITERATIONS=86400` in `reference/compose/.env`.
   - That contradicts the implementation in this branch: `.env.example` drops the env var, compose no longer passes it, and the effective knob is now `max_quiescent_iterations` in the experiment config YAML. The new config-field table earlier in the same doc is correct; these older sections were left behind.
