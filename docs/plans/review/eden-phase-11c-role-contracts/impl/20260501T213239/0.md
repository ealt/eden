## Change Summary

- **20 new conformance scenarios** in three new files (4 planner, 7 implementer, 9 evaluator), each citing a chapter-03 MUST and grouped under three new chapter-9 §5 v1+roles index entries. Brings the suite from 86 to 106 tests.
- **Spec amendment to §4.4** (chapter 03): tightened evaluator-resubmit-equivalence to drop `artifacts_uri`, with rationale that chapter-04 §4.2 is canonical and the role-side restatement defers to it. Surfaced via codex-review round 0 on the plan; impl docstring at `submissions.py:65` already followed §4.2.
- **Helper extensions in `_seed.py`**: `submit_plan` now always sends `proposal_ids` (was omitted on `status="error"`, which is non-conforming under the literal-spec reading of §2.4). `submit_evaluate` accepts explicit `metrics` regardless of `status` and gains an `artifacts_uri` parameter, so eval_error-with-metrics + vary-artifacts_uri tests drive real wire payloads instead of silently no-op'ing.
- **End-state-not-endpoint pattern** for two §3.4 / §4.2 tests where the IUT has latitude on where to reject (submit-time vs accept-time) — the test asserts the observable end-state (`trial.status != "success"`) rather than pinning the failing status code at one specific endpoint. Documented in plan §"The 'assert end-state, not endpoint' pattern" so 11d's integrator scenarios inherit the same discipline.
- **Three new tests added in plan-review iterations** (rounds 1-2 of plan codex-review): implementer status=error round-trip + no-evaluate-task-dispatched (§3.4 second sentence), evaluator status=error trial-side metrics + artifacts_uri write (§4.4 per-status write rule), evaluator vary-only-artifacts_uri-resubmit accepted (pins the §4.4 amendment).
