- Bug — [test_implementer_submission.py](/Users/ericalt/Documents/eden/conformance/scenarios/test_implementer_submission.py:96): `test_success_without_commit_sha_must_not_complete_trial` treats any non-`200` submit response as “conforming rejection” and any non-2xx `/accept` response as acceptable. - Risk — [_seed.py](/Users/ericalt/Documents/eden/conformance/src/conformance/harness/_seed.py:304): `drive_to_starting_trial()` does not check whether the implementer `/accept` succeeded before returning a trial that callers assume is ready for evaluator scenarios.
