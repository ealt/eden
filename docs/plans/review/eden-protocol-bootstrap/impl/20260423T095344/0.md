## Change Summary

- **Three spec chapters** land under `spec/v0/`:
  - `00-overview.md`: scope, conformance classes (planner,
    implementer, evaluator, integrator, task store, event log,
    artifact store, orchestrator), version lineage policy, document
    conventions.
  - `01-concepts.md`: protocol vocabulary — experiment, roles,
    proposal, trial, task, event, objective/metrics-schema, claim
    token, canonical trial lineage (`main` / `trial/*` / `work/*`),
    storage components, orchestrator. Intentionally concept-level; no
    state-machine semantics (those land in Phase 2).
  - `02-data-model.md`: entity-by-entity field spec with required/
    optional split, JSON-level types, cross-entity structural
    invariants. Each entity cross-references its JSON Schema.
- **Six JSON Schemas** under `spec/v0/schemas/`, Draft 2020-12. `$id`
  omitted so cross-schema `$ref`s resolve against the local
  filesystem (simplifies local validation without a hosted resolver).
  `experiment-config.schema.json` `$ref`s `metrics-schema.schema.json`
  by relative filename.
- **Fixture migrated** verbatim from
  `direvo/tests/fixtures/experiment/.eden/config.yaml`; validates
  against the experiment-config schema as-is.
- **`schema-validity` CI job** added. Uses
  `check-jsonschema==0.29.4`, pinned. Two steps:
  1. `--check-metaschema spec/v0/schemas/*.schema.json` validates each
     schema file against the Draft 2020-12 meta-schema.
  2. `--schemafile experiment-config.schema.json <fixture>` validates
     the fixture against the experiment-config schema.
- **AGENTS.md commands table** adds two `pipx run` commands (same
  pinned version as CI) and a new "Phase 1 in progress" status.
- **README.md status** updated to "Phase 1 in progress" with the
  branch-awaiting-review framing.
