## Change Summary

- `05-event-protocol.md` pins the envelope, the transactional
  invariant (atomic event+state), the closed registry of 15 v0 event
  types (6 task / 4 proposal / 5 trial), payload shapes per type,
  composite-commit cases enumerated (e.g. `task.created` +
  `proposal.dispatched` for implement dispatch), and delivery
  guarantees (per-experiment total order, causal consistency,
  at-least-once, replayability, durability).
- `event.schema.json` now carries `$defs` for shared types
  (`taskId`, `commitSha`, `failReason`, `reclaimCause`, …) and an
  `allOf` array of 15 `if/then` clauses that dispatch on `type` to
  enforce per-type `data` payload shapes. Unregistered types pass
  the envelope only (§3.5 open extension).
- `06-integrator.md` pins: three-namespace topology with `main`
  immutability; sole-integrator rule for `trial/*`; promotion
  trigger (`status==success`, `commit_sha` set,
  `trial_commit_sha` absent); squash rule (single commit with
  `tree(T)==tree(commit_sha)` and parents from proposal's
  `parent_commits`); commit-message format (`trial: <trial_id>
  <slug>`); atomic-three invariant (ref write + `trial_commit_sha`
  field + `trial.integrated` event); eval manifest at
  `.eden/trials/<trial_id>/eval.json` with required fields
  (trial_id, proposal_id, commit_sha, parent_commits, metrics,
  artifacts, evaluator, evaluated_at); artifacts entries include
  `sha256` and `bytes` for reproducibility.
- `08-storage.md` specifies the three stores. Task store: operation
  table, atomic claim (linearizable), token-authorization rule,
  idempotent resubmit, terminal immutability, per-task
  serialization. Event log: append / read_range / subscribe /
  replay operations, ordering + causality, at-least-once delivery,
  retention floor. Durability: write durability, read-after-write
  consistency, crash recovery, no-fabrication. Per-experiment
  metrics-schema enforcement. Artifact store: upload/fetch,
  durability tied to retention, content-integrity (no overwrite
  under a referenced URI). Transactional guarantees consolidated
  for composite commits (§6).
- `event.py` adds `RegisteredEvent = Annotated[... | ..., Field(
  discriminator="type")]` with one model per registered type, a
  `RegisteredEventAdapter`, and a `REGISTERED_EVENT_TYPES` frozenset
  for the test harness to dispatch on. Each per-type payload is its
  own private `BaseModel` (`_TaskCreatedData`, `_ProposalAndTaskData`,
  …). Optional fields still follow the `NotNone` discipline from
  Phase 3; but registered payloads here are all-required at v0.
- Round-trip and schema-parity tests now dispatch through the typed
  registered-event path when applicable, so a regression that
  dropped per-type payload enforcement would fail CI.

## Round 1 — changes in response to round-0 feedback

### Bug 1 (fixed): squash rule vs eval-manifest tree contradiction

`spec/v0/06-integrator.md` §3.2 rewritten. The tree invariant is now:
`tree(T)` equals `tree(commit_sha)` with exactly one path added —
the eval-manifest file at `.eden/trials/<trial_id>/eval.json`. No
other path may be added, removed, or modified. Also added: the
implementer is not required to carry the manifest; if a worker
branch happens to already contain a file at the manifest path, the
integrator MUST reject the trial (no `trial/*` commit, no
`trial.integrated` event).

### Bug 2 (fixed): eval-manifest fields with no protocol source

`spec/v0/06-integrator.md` §4.2, §4.3, §4.4 restructured. Required
fields reduced to only those written through the existing role
contracts in `03-roles.md`: `trial_id`, `proposal_id`, `commit_sha`,
`parent_commits`, `metrics`, `completed_at` (using the trial's
already-written `completed_at` rather than adding an `evaluated_at`
field). Optional fields: `artifacts_uri`, `description`,
`evaluator`, `artifacts`. The `evaluator` object (§4.3) and the
per-file `artifacts` inventory with `sha256`/`bytes` (§4.4) are now
explicitly optional — populatable when a deployment's role-binding
surfaces that metadata. Kept as optional rather than dropped so
deployments wanting stronger reproducibility still have a normative
place to record it; SHOULD-prefer the inventory when present.

### Bug 3 (fixed): overclaim in §3.4 "reconstruction guarantee"

`spec/v0/05-event-protocol.md` §3.4 renamed to "Subscriber
**observability** guarantee" and rewritten. A subscriber consuming
every registered event in log order reconstructs the **lifecycle
history** (which entities exist, in what states, in what order);
content retrieval (`parent_commits`, `metrics`, etc.) is via the
respective stores using the identifiers the events carry. The
transactional invariant (§2) ensures any entity reachable via an
event identifier is durable at read time. No schema or Pydantic
change was needed — the bindings were already in lockstep with the
minimal registry; only the spec prose was overclaiming.

### Risk 4 (fixed): proposal/trial persistence not covered

`spec/v0/08-storage.md` gained a new §1.7 "Proposal and trial
persistence". Required operations (Create, Read, List, Update);
writes MUST commit atomically with their events per §6; durability,
read-after-write, crash-recovery, no-fabrication apply uniformly;
terminal immutability applies (proposals in `completed`, trials in
`success`/`error`/`eval_error`). The `trial_commit_sha` field is
called out as the **one** post-terminal write permitted, belonging
to the integrator and committed atomically with its event and ref.

### Risk 5 (fixed): replayability lifetime ambiguity

`spec/v0/05-event-protocol.md` §4.4 and `spec/v0/08-storage.md` §2.4
now carry the same contract: for the lifetime of an experiment —
from registration until operator-declared terminal state — a
conforming log MUST retain every appended event and serve full
replay from the first event; no compaction during that window. After
terminal state: compaction MAY occur, replays older than the horizon
MUST be served from an archive that preserves order/content or
rejected with a well-defined error, never silently dropped.
"Operator-declared terminal state" is forward-referenced to Phase 12
with a v0 guidance note.
