## Change Summary

- **Protocol-first framing wired through every doc.** Root README,
  AGENTS, CONTRIBUTING, STYLE_GUIDE, and every section README describe
  the split: `spec/` is authoritative, `reference/` is one valid impl,
  `conformance/` is the suite. CONTRIBUTING has two paths (spec vs
  reference impl) with different discipline.
- **Roadmap has units + chunks.** `docs/roadmap.md` enumerates Phases
  0–13 with fine-grained named units (1a, 1b, 1c, …) and explicit
  chunk-grouping annotations that describe which units travel together
  due to design coupling. Phases 0–11 have full detail; 12 and 13 are
  placeholders. Each phase has explicit non-goals to prevent creep.
- **Spec v0 is scaffold-only.** `spec/v0/README.md` tabulates the 10
  planned chapters and 6 planned schemas with target-phase annotations;
  no chapter content or schema files exist yet. That is deliberate —
  writing them is Phase 1.
- **Reference is all .gitkeep.** Every service and package directory
  exists under `reference/` but contains only a `.gitkeep`.
  `reference/README.md` tabulates which component lands in which phase.
- **CI is one job, docs-only.** `.github/workflows/ci.yml` has a single
  `docs-lint` job running `markdownlint-cli2` pinned to 0.14.0. No
  Python jobs (no Python code yet). Exclusions: `node_modules`, `.venv`,
  `docs/archive/**`, `docs/plans/review/**` (the last added after an
  initial lint run showed Codex review transcripts would fail standard
  rules — they're tool output, not authored prose).
- **Local lint passes.** The exact CI command runs clean locally (0
  errors, 12 files linted) — duplicated heading fixes in
  `CONTRIBUTING.md`, language tags on code fences in `naming.md` and
  `eden-protocol-bootstrap.md`.
