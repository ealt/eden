**Findings**

1. Major: the integrate-over-HTTP story is much better, but it still treats “read-back shows no `trial_commit_sha` after bounded retries” as proof that the server did not commit. That is not actually a positive confirmation; the original request can still be in flight and commit just after the last read, which would recreate the ref-deleted / store-committed split you were trying to eliminate. The safe rule is stricter: if reconciliation observes the expected SHA, return success; if it observes a different SHA, raise loudly; if it cannot definitively prove non-commit, raise `IndeterminateIntegration` and do not compensate. As written, steps `b`/`c` and the “positively confirmed” claim overstate what the bounded read-back loop can establish. 2. Major: the new retry guidance for “other mutations” is too broad and is incorrect for at least `claim`. An `illegal-transition` on retry does not generally mean “the prior call committed”; it can also mean some other actor won the race or the state changed for another reason. Even in the happy case, `claim` still has the unresolved token-recovery problem. I would not standardize this as a binding convention. If you want retry guidance here, it needs to be per-operation and based on a validating follow-up read, not a blanket reinterpretation of `IllegalTransition`. Ref: [eden-phase-8a-wire-protocol.md](/Users/ericalt/Documents/eden/docs/plans/eden-phase-8a-wire-protocol.md:181)

3. Medium: the plan still leaves the different-SHA `integrate_trial` error type unresolved in one place, but later tests assume a specific answer. The normative section says the chapter will pick either `conflicting-resubmission` or `invalid-precondition`, while the test section hard-codes `invalid-precondition`. That needs to be settled in the plan text before implementation or you’ll get drift between chapter, client mapping, and tests. The algorithm section still calls `GET /tasks/{id}/validate-terminal` instead of the new `/_reference/...` path, and the package section still says “see Open questions” even though that section was renamed. **Assessment**

The `subscribe` binding now looks complete for 8a, and the normative vs. `/_reference/` split is clean. Moving helper endpoints out of `07-wire-protocol.md`, switching `InvalidPrecondition` to `409`, and adding explicit long-poll tests are all solid fixes.

I would still hold the plan until the integrate reconciliation rule is tightened and the blanket retry convention for non-idempotent mutations is removed or narrowed. After that, the design is in much better shape.
