**Findings**
- Control-plane worker auth is still incompatible with 12a-1’s per-experiment worker/credential scope. [eden-phase-12c-control-plane.md](/Users/ericalt/Documents/eden/docs/plans/eden-phase-12c-control-plane.md:221) lines 221-239 make lease ops authenticate with a worker credential on global `/v0/control/*` endpoints, and [eden-phase-12c-control-plane.md](/Users/ericalt/Documents/eden/docs/plans/eden-phase-12c-control-plane.md:347) lines 347-353 bootstrap the orchestrator once before leasing arbitrary experiments. But 12a-1 made workers local to an experiment, not deployment-global: [eden-phase-12a-1-worker-identity.md](/Users/ericalt/Documents/eden/docs/plans/eden-phase-12a-1-worker-identity.md:24) lines 24-27. As written, there is no unambiguous registry against which the control plane validates a worker credential for `list_experiments` or `acquire_lease(E)` across many experiments. This needs either deployment-scoped control-plane identities, or an explicit “register same orchestrator worker in every experiment” model with corresponding bootstrap and credential rules.

- The Option-A registry still mirrors `last_known_state` without any synchronization path. [eden-phase-12c-control-plane.md](/Users/ericalt/Documents/eden/docs/plans/eden-phase-12c-control-plane.md:265) lines 265-276 make `last_known_state` authoritative enough to gate `unregister_experiment`, and the dashboard shows experiment state at [eden-phase-12c-control-plane.md](/Users/ericalt/Documents/eden/docs/plans/eden-phase-12c-control-plane.md:539) lines 539-541.
