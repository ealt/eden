**Remaining Substantive Issues**

- Work-ref ownership is still too SHA-centric. The GC logic cross-references refs against trials by `commit_sha`, but the canonical ref name already embeds the owning `trial_id` as `work/<slug>-<trial_id>`. If two trials ever share a commit SHA, or if a ref name and SHA disagree due to manual rewrite, this plan can associate the ref with the wrong trial and offer deletion on the wrong basis. The safer rule is: parse the `trial_id` from the ref name first, load that trial, then require the ref’s current SHA to match that trial’s `commit_sha`. See [eden-phase-9e-observability-and-admin.md](/Users/ericalt/Documents/eden/docs/plans/eden-phase-9e-observability-and-admin.md:373), [implementer.py](/Users/ericalt/Documents/eden/reference/services/web-ui/src/eden_web_ui/routes/implementer.py:69), [proposal.py](/Users/ericalt/Documents/eden/reference/packages/eden-contracts/src/eden_contracts/proposal.py:18).
- The trial-detail event ordering should not permit sorting by `event_id`. The plan says `event_id` or replay index are both monotonic, but `event_id` comes from a pluggable factory and is not a reliable ordering contract. Use replay order / natural index only. See [eden-phase-9e-observability-and-admin.md](/Users/ericalt/Documents/eden/docs/plans/eden-phase-9e-observability-and-admin.md:316), [_base.py](/Users/ericalt/Documents/eden/reference/packages/eden-storage/src/eden_storage/_base.py:166).
- The read-path 404 behavior is still underspecified. The plan says `NotFound` on read paths goes through the existing 404 handler, but the app only has a handler for HTTP 404 responses, not `eden_storage.NotFound` / wire `NotFound` exceptions. Either the routes need to catch and translate those explicitly, or `make_app` needs an exception handler for the storage/wire exception class. See [eden-phase-9e-observability-and-admin.md](/Users/ericalt/Documents/eden/docs/plans/eden-phase-9e-observability-and-admin.md:527), [app.py](/Users/ericalt/Documents/eden/reference/services/web-ui/src/eden_web_ui/app.py:90), [errors.py](/Users/ericalt/Documents/eden/reference/packages/eden-wire/src/eden_wire/errors.py:92).
