## Change Summary

- **Default overlay** tails the #109 per-service JSONL bind-mount (`${EDEN_EXPERIMENT_DATA_ROOT}/logs/**/*.jsonl`) read-only via Alloy → Loki; **no docker socket** (lower privilege than the issue's `docker_sd_configs` sketch, which #109 made unnecessary). Loki + Alloy internal-only; Grafana the only host-exposed surface (`${GRAFANA_HOST_PORT:-3000}`). Images pinned: `grafana/loki:3.7.2`, `grafana/alloy:v1.16.1`, `grafana/grafana:12.4.3`.
- **Alloy, not Promtail** (Promtail EOL 2026-03-02). Pipeline: `local.file_match` → `loki.source.file` → `loki.process` (json/timestamp/labels, lifting service/level/experiment_id to labels, record `ts` as entry time via RFC3339Nano) → `loki.write`.
- **Optional infra overlay** (`compose.logging-infra.yaml`) layered on top adds postgres/forgejo **stdout** capture via Alloy's `discovery.docker` + `loki.source.docker`. It mounts the docker socket, so it is isolated in its own overlay (AGENTS.md privilege-isolation discipline) and uses a dedicated `:?`-guarded `EDEN_LOGGING_DOCKER_GID` (NOT `EDEN_DOCKER_GID`, which defaults to 0).
- **Storage classification:** `loki/` + `alloy/` data-root dirs are DERIVED / non-protocol-owned (chapter 01 §13) — a queryable projection of `logs/`, not durable state.
- **setup-experiment** always generates `EDEN_GRAFANA_ADMIN_PASSWORD` (idempotent) + creates `loki/`+`alloy/` dirs (chmod 0777; Loki uid 10001). No `--with-logging` flag.
- **smoke-logging.sh** (bash 3.2-safe) + `compose-smoke-logging` CI job (path-gated, non-required). Validated end-to-end on a live bring-up incl. the infra overlay.

## Round 1 — fixes applied

**Finding 1 (Risk) — infra streams excluded from the starter dashboard. ADDRESSED.**
Root cause confirmed: the panel selector `{experiment_id=~"$experiment_id", service=~"$service", level=~"$level"}` requires the `experiment_id` + `level` labels, which the infra-overlay docker-stdout streams (postgres/forgejo) lack, so the default "All" view excluded them (verified live: with the old value-list All, `label_values(experiment_id)` returns only EDEN ids, so `experiment_id=~"(id)"` drops label-less streams). This also violated the project's "observability must not hide data by default" principle.
Fix: set template-variable `allValue` so the default view does not exclude label-less streams — `service` → `.+` (the load-bearing non-empty matcher Loki requires; every stream has `service`), `experiment_id` + `level` → `.*` (match streams missing the label). Validated live: the exact default selector `{experiment_id=~".*", service=~".+", level=~".*"}` returns BOTH orchestrator (50) and postgres (13) lines. Loki accepts it (the `.+` on service is the non-empty anchor; an all-`.*` selector is rejected). Documented the behavior in `observability.md` §2.8. Regression-guarded in `smoke-logging.sh` (Phase 5 asserts the default selector returns EDEN lines; Phase 6 asserts it also surfaces infra streams).

**Finding 2 (Risk) — smoke only proved `{service="postgres"}`. ADDRESSED.**
`smoke-logging.sh` Phase 6 now also polls + asserts `{service="forgejo"}` ≥ 1 (verified live: forgejo emits startup stdout even at log level Warn — 5–9 lines). A relabel-rule / docker-source regression that drops forgejo now fails the smoke.

**Finding 3 (Nit) — file-tail pipeline duplicated across the two Alloy configs. ADDRESSED via coverage (duplication kept, deliberately).**
The duplication is retained: Alloy's clean composition story for a *single bind-mounted config file* is weaker than the cost of the ~40-line duplication, and the default overlay must keep a socket-free standalone config. Instead, per Codex's alternative ("strengthen coverage so both configs are exercised against the same label expectations"), `smoke-logging.sh` Phase 6 now re-asserts `{service="orchestrator"}` still returns *under the infra config* — so drift in the duplicated file-tail half of `alloy-config-infra.alloy` can no longer stay green. The "keep in sync" header comment is retained.

**Validation:** `smoke-logging.sh` re-run end-to-end — PASS (postgres 5, forgejo 5, EDEN-under-infra OK, dashboard default selector surfaces both). markdownlint / rename-discipline / shellcheck / `bash -n` clean.
