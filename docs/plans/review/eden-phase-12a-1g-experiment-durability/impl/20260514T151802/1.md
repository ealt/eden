## Change Summary

- **Spec.** §13 "Experiment durability" added to chapter 01 (anchored
  to §1, not replacing it). Cross-references in chapter 08 §3 and
  chapter 09 §5. The §13 has multiple MUSTs (aggregate-survival;
  no-loss-as-side-effect-of-maintenance) so the citation check
  resolves naturally.

- **Compose binding.** 12 durable named volumes → bind-mounts under
  ${EDEN_EXPERIMENT_DATA_ROOT}/<subdir>. eden-blob-data + blob-init
  service removed entirely (no consumer; grep audit clean). The
  subprocess overlay was audited for short-form duplicates of base-
  file durable mounts and pruned. The docker-exec overlay's
  --exec-volume flags for durable substrates rewrote to --exec-bind
  so sibling DooD containers mount the same host paths the worker
  host sees.

- **setup-experiment.** --data-root flag with : rejection at flag-
  parse; idempotent preservation across re-runs; abort-on-
  incompatible-relocation guard; substrate tree mkdir + chmod 0777
  (same cross-uid posture as the existing cidfile dir precedent).

- **Smokes.** All four scripts use a per-script mktemp -d + trap rm
  -rf EXIT and pass --data-root to setup-experiment. The smoke.sh
  substrate assertion changed from docker-volume-inspect to subdir-
  exists. blob-data refs removed from cleanup loops.

- **Operator doc.** New docs/operations/experiment-data-durability.md
  covers the substrate layout, durability posture matrix (what
  survives / what doesn't), custom data root, permissions, migration
  recipe from named volumes, and a self-contained manual kill-and-
  restart verification recipe.
