## Change Summary

- Added 9 new files (1 routes module, 7 templates, 1 admin tests-flow file with 5 supporting test files actually — see table) under `reference/services/web-ui/`. `routes/admin.py` is the load-bearing addition (~430 lines).
- Modified `app.py` to register the admin router and add a `StorageNotFound` exception handler so raised storage/wire `NotFound` gets the 404 page (the existing handler only caught HTTP 404 *responses*).
- Added the permanent "admin" nav link in `base.html` (after evaluator).
- Reused the existing `eden_git.GitRepo.delete_ref(refname, *, expected_old_sha=…)` primitive — no `eden-git` changes.
- 63 new tests, 0 changes to existing tests; full suite is 625 passing (including the e2e and the parametrized security cases).
- ruff/pyright/markdownlint clean across the whole repo.
- Auth-first POST discipline matches the existing planner / implementer / evaluator modules: `get_session` runs before `csrf_ok`; unauthenticated POSTs redirect to `/signin` (303), missing-CSRF returns 403.
- Work-ref ownership keys off **exact `trial.branch` equality** per round-2 codex feedback, not by parsing `work/<slug>-<trial_id>` (which fails when `trial_id` itself contains `-`, e.g. `trial-<hex>`).
- Trial-event correlation is the algorithm pinned in §A.5: union of `event.data.trial_id == trial_id` matches plus task-id matches for the implement task that produced this trial and any evaluate task referencing it, ordered by replay-index position.
- Event-log view computes natural index *before* slicing/filtering/reversing so the rendered position is stable across all transformations.
