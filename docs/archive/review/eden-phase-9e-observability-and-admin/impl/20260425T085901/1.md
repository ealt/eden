# Implementation Review: eden-phase-9e-observability-and-admin (Round 1)

## Round 0 findings — addressed

1. **Bug — read-path transport handling missing.** Fixed: every admin GET handler that calls `store.*` / `repo.*` now wraps the read in `try / except Exception` and renders an inline `_error.html` placeholder via the new `_read_failure_response(request, message)` helper. `StorageNotFound` is re-raised so the app-wide 404 handler picks it up. Routes touched: `index`, `tasks_index`, `task_detail`, `trials_index`, `trial_detail`, `events_index`, `work_refs_index`. New tests in `test_admin_partial_write.py::TestReadPathTransportFailures` cover the index, tasks, and events read paths under `monkeypatch InMemoryStore.list_tasks` / `replay`.

2. **Bug — `work_refs_delete()` collapsed every git failure into `?error=ref-changed`.** Fixed: route now classifies `GitError` based on stderr containing `expected` (or returncode 1, the CAS-miss surface) → `?error=ref-changed`; any other `GitError` (e.g. permission denied, returncode 128) re-raises so FastAPI's default 500 handler renders. Test `TestWorkRefDeleteFailures::test_unexpected_git_error_propagates_5xx` exercises this with a `returncode=128` permission-denied stderr.

3. **Risk — `_coerce_filter()` mapped unknown filter values to `None` (full list).** Fixed: introduced `_INVALID_FILTER` sentinel; routes short-circuit to an empty rowset when the filter is invalid. Test `test_unknown_kind_renders_empty` and `test_unknown_state_renders_empty` lock in the new behavior; the prior `test_unknown_kind_treated_as_no_filter` (which codified the wrong contract) was renamed and its assertions inverted.

4. **Risk — dashboard collapsed "no `--repo-path`" and "git failed" into the same `None`.** Fixed: route now passes both `repo_enabled: bool` and `work_ref_error: bool` flags to the template. `admin_index.html` renders three distinct states: "not configured" / "git failure listing refs" (banner-error) / "{N} ref(s)" with the GC link.
