# Implementation Review: EDEN Phase 3 — Round 2 update

## Round 1 feedback addressed

### Bug: URI "has a scheme" check not RFC 3986

**Addressed.** Replaced the `urlparse`-based check in `_common.py` and in `conftest.py` with `rfc3986_validator.validate_rfc3986(v, rule="URI")` — the canonical JSON Schema URI format validator. Added `rfc3986-validator>=0.1.1` as a runtime dependency of `eden-contracts` so the model-side check uses the same implementation as the parity-side `FormatChecker`.

Spot-checked the validator directly:

```text
'http://exa mple.com/path'  -> None (rejected)
'http://example.com/bad%ZZ' -> None (rejected)
'http://example.com'        -> match  (accepted)
's3://bucket/path'          -> match  (accepted)
'file:///tmp/x'             -> match  (accepted)
'just a string'             -> None (rejected)
```

New reject cases:

- `proposal::artifacts_uri_space_in_host` — `http://exa mple.com/path`
- `proposal::artifacts_uri_bad_percent_encoding` — `http://example.com/bad%ZZ`
- `trial::artifacts_uri_space_in_host`

All 78 tests pass.
