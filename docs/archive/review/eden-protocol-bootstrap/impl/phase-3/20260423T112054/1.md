# Implementation Review: EDEN Phase 3 — Round 1 update

## Round 0 feedback addressed

### Bug: numeric coercion (config.py, proposal.py)

**Addressed.** Every top-level model now sets `model_config = ConfigDict(strict=True, extra="allow")`. In Pydantic 2 strict mode this rejects bool → int, str → int, str → float, and bool → float coercions while still allowing int → float (which JSON number semantics permit). New corpus cases now assert this directly:

- `experiment-config::parallel_trials_bool`
- `experiment-config::max_trials_string`
- `experiment-config::convergence_window_string`
- `proposal::priority_bool`
- `proposal::priority_string`

All six schema-parity + roundtrip tests over them pass.

### Bug: artifacts_uri not format-validated (proposal.py, trial.py)

**Addressed.** Introduced `UriStr` in `_common.py` — `Annotated[str, AfterValidator(_check_uri)]` — where `_check_uri` uses `urlparse` and rejects any value without a scheme. Applied to `Proposal.artifacts_uri` and `Trial.artifacts_uri`. The parity-side validator now carries a custom `FormatChecker` that enforces the same `uri` format semantics, so schema and model agree.

New reject cases:

- `proposal::artifacts_uri_no_scheme`
- `trial::artifacts_uri_no_scheme`

### Bug: date-time regex accepts impossible values (_common.py)

**Addressed.** `DateTimeStr` is now `Annotated[str, StringConstraints(pattern=...), AfterValidator(_check_datetime)]` where `_check_datetime` calls `datetime.fromisoformat`. This catches impossible dates and times after the regex pre-filter. The parity `FormatChecker` does the same on the schema side.

New reject cases:

- `task::impossible_datetime`
- `proposal::created_at_impossible`

### Risk: parity job too weak (conftest.py, cases.py)

**Addressed.** Two independent changes:

1. `conftest.py` now passes a custom `FormatChecker` to the validator, so schema-side `format: uri` and `format: date-time` checks actually run. Without this, the schema validator silently ignored `format` and the parity test could hide real drift.
2. `cases.py` is expanded to cover the exact gaps Codex flagged: bool/string numerics, invalid URIs, impossible timestamps, explicit null on optional fields. 75 cases now (up from 61).

### Risk: explicit null on optional fields (task.py, config.py, trial.py)

**Addressed.** Added `NotNone = BeforeValidator(_reject_none)` in `_common.py` and applied it to every optional field across Trial, Task, TaskClaim, ExperimentConfig. By default Pydantic does not run validators on a field's default value (`validate_default=False`), so an omitted field still uses its default; only an explicit `null` in input triggers the validator, which raises. Net effect: absent is accepted, explicit null is rejected — matching JSON Schema semantics for optional typed fields.

New reject cases:

- `experiment-config::convergence_window_null`
- `task::claim_null_on_pending`
- `trial::metrics_explicit_null`
- `trial::completed_at_null`

### Nit: stale phase-status prose (AGENTS.md, CONTRIBUTING.md)

**Addressed.** Folded the status flip into this PR rather than deferring it to a follow-up. `AGENTS.md` and `CONTRIBUTING.md` now read "Phase 3 complete" and `README.md` is refreshed with a description of what the contracts package and CI gates actually provide. This replaces the two-PR pattern used in Phase 1/2 because the documentation edits are substantive in this PR, not just a header flip.

## Deliberately not changed

- **Pyright typeCheckingMode.** Still `standard`, not `strict`. Tightening Phase 3 further adds noise on Pydantic patterns without catching the protocol-parity defects this PR is about. Easier to revisit once there's more code consuming the contracts.
- **Pydantic `AnyUrl` vs custom `UriStr`.** I kept a custom `str`-returning validator rather than adopting `pydantic.AnyUrl`, because `AnyUrl` stores a structured `Url` object and rewrites input (e.g., adds trailing slashes) in ways that break byte-for-byte round-trip. A plain string plus a schema-parity-equivalent `_check_uri` keeps the model's serialized form identical to the input, which is what the parity test requires.
- **Auto-discovering workspace members.** Still an explicit list (`["reference/packages/eden-contracts"]`) rather than a `reference/packages/*` glob. The glob would pick up the Phase 0 placeholder directories. The alternative — seeding a stub `pyproject.toml` in every placeholder — is more work for more code.
