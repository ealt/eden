## Change Summary

- **PostgresStore + psycopg v3.** Third backend conforming to the same `Store` Protocol as `InMemoryStore`/`SqliteStore`. Schema mirrors SQLite byte-for-byte (`data text NOT NULL`, not `jsonb`). Connection runs `autocommit=True` with explicit per-op `BEGIN ISOLATION LEVEL SERIALIZABLE READ WRITE` (psycopg's default auto-begin would error `ActiveSqlTransaction` on `BEGIN ISOLATION LEVEL`). Per-test schema isolation in `conftest.py` (`CREATE SCHEMA test_<rand>` + `search_path` override + `DROP SCHEMA … CASCADE` teardown).
- **`--store-url` URL dispatch.** `task-store-server` now takes `--store-url` (`:memory:` / `sqlite:///<path>` / `postgresql://…` / bare-path-for-compat). The deprecated `--db-path` alias survives one phase with a logged deprecation. Mass-rename across the existing e2e tests.
- **Shared `eden-reference:dev` image.** Multi-stage Dockerfile: builder runs `uv sync --frozen --no-dev --all-packages` (the `--all-packages` flag is load-bearing because `eden-*` packages live in the `dev` group of the non-installable workspace root), runtime stage copies `/app` and pre-creates `/var/lib/eden/{repo,artifacts}` with `eden:eden` ownership so named-volume mounts inherit the right perms. New top-level `.dockerignore` keeps `.git`, `__pycache__`, etc. out of the build context.
- **Compose service entries** for the six EDEN services + a `setup`-profiled `eden-repo-init` one-shot. Healthchecks for `task-store-server` (authenticated GET against the experiment's events endpoint) and `web-ui` (unauthenticated `/healthz` added to `app.py`). Orchestrator uses `restart: "on-failure"` (so exit-0 on quiescence sticks) and Compose-friendly tuning (`--poll-interval 1.0 --max-quiescent-iterations 30`). Two new named volumes: `eden-bare-repo` (mounted by orchestrator/implementer/web-ui) and `eden-artifacts-data` (web-ui). Compose `configs:` for the experiment-config YAML.
- **`setup-experiment.sh`** — bash script that: validates tooling, parses CLI args, generates/preserves secrets in `.env`, copies the experiment config into `reference/compose/`, runs `docker compose build eden-repo-init` + `docker compose run --rm --no-deps eden-repo-init`, parses the seed SHA from stdout via `sed -nE`, and *replaces* the placeholder `EDEN_BASE_COMMIT_SHA=000…000` line in `.env` with the real SHA (replace-not-append so `grep ^X=` returns one definition).
- **`eden_service_common.repo_init` module.** Idempotent: short-circuits if `/var/lib/eden/repo/HEAD` exists, otherwise runs `git init --bare --initial-branch=main` + `seed_bare_repo`. Emits `EDEN_REPO_SEEDED sha=<hex>` or `EDEN_REPO_ALREADY_SEEDED sha=<hex>`.
- **Extended `compose-smoke` CI** that drives setup-experiment, asserts the seeded ref before `compose up` via a one-shot probe that reuses `eden-reference:dev` (avoids floating `latest` tag), waits up to 180s for the orchestrator to exit 0 on quiescence, and verifies ≥3 `trial.integrated` events on the wire endpoint. New `python-test-postgres` CI job runs the parametrized backend tests against a postgres service container.

- **Healthcheck variable interpolation.** I deliberately use single-`$` Compose-load-time interpolation (e.g., `${EDEN_SHARED_TOKEN}`) rather than container-shell `$$` because the env-file vars aren't in the container's shell environment unless declared under `environment:`; baking values into the literal command at compose-load is simpler and avoids a redundant `environment:` block.
- **`web-ui` repo mount is read-write,** not read-only — `--repo-path` activates the entire implementer module which writes refs on submit. Documented in the matrix and the Compose README. Operators get both the unattended `implementer-host` and the human-driven web-ui implementer module; they race for claim atomicity via `Store.claim`.
