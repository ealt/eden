**Findings**

- Bug — [reference/services/web-ui/src/eden_web_ui/routes/evaluator.py](/Users/ericalt/Documents/eden/reference/services/web-ui/src/eden_web_ui/routes/evaluator.py:200): the submit handler only copies metric fields already present in `metrics_schema`, so a hand-crafted `metric.ghost` field is silently dropped before `parse_evaluate_form` sees it. **Overall Assessment**

The implementation is mostly aligned with the phase 9d plan, and the `IllegalTransition -> read-back` recovery logic is implemented correctly. The main substantive issue is the missing route-level rejection of unknown metric keys; that is both a plan-adherence miss and a real correctness bug. The other gaps are secondary: transport-read failures still surface as 500s, the test suite overstates coverage for the unknown-metric case, and one minor template behavior from the plan is missing.
